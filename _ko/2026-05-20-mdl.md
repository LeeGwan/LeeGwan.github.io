## [0x00] 개요

MDL을 공부한 이유는 `CR0`의 `WP`(Write Protect) 비트를 비활성화하기 위해서이고,
이는 커널 네이티브 함수에 인라인 후킹을 하기 위한 선행 작업입니다. 이 글은
코드, 표, 목차가 어떻게 렌더링되는지 보여주는 예시입니다.

## [0x01] 구조체

`_MDL` 구조체는 WinDbg에서 다음과 같이 확인할 수 있습니다:

```
0: kd> dt nt!_MDL
   +0x000 Next             : Ptr64 _MDL
   +0x008 Size             : Int2B
   +0x00a MdlFlags         : Int2B
   +0x028 ByteCount        : Uint4B
   +0x02c ByteOffset       : Uint4B
```

MDL을 할당하고 잠그는 간단한 드라이버 엔트리입니다:

```c
NTSTATUS DriverEntry(PDRIVER_OBJECT pDriver, PUNICODE_STRING pRegPath)
{
    PMDL pMDL = IoAllocateMdl(buffer, 0x10, FALSE, FALSE, NULL);
    MmProbeAndLockPages(pMDL, KernelMode, IoReadAccess);
    /* ... */
    MmUnlockPages(pMDL);
    IoFreeMdl(pMDL);
    return STATUS_SUCCESS;
}
```

## [0x02] 메모

| 멤버 | 의미 |
|--------|---------|
| StartVa | 가상 버퍼의 페이지 정렬된 시작 주소 |
| ByteOffset | 그 페이지 안에서 실제 VA의 오프셋 |
