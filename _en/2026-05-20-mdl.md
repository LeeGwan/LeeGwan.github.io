## [0x00] Overview

I studied MDLs in order to disable `WP` (Write Protect) in `CR0`, which is a
prerequisite for inline-hooking native kernel functions. This post is a sample
showing how code, tables, and the table of contents render.

## [0x01] The structure

The `_MDL` structure can be dumped in WinDbg:

```
0: kd> dt nt!_MDL
   +0x000 Next             : Ptr64 _MDL
   +0x008 Size             : Int2B
   +0x00a MdlFlags         : Int2B
   +0x028 ByteCount        : Uint4B
   +0x02c ByteOffset       : Uint4B
```

A simplified driver entry that allocates and locks an MDL:

```c
NTSTATUS DriverEntry(PDRIVER_OBJECT pDriver, PUNICODE_STRING pRegPath)
{
    PMDL pMDL = IoAllocateMdl(buffer, 0x10, FALSE, FALSE, NULL);
    MmProbeAndLockPages(pMDL, KernelMode, IoReadAccess);
    /* ... */
    MmUnlockPages(pMDL);
    IoFreeMdl(pMDL);
    return STATUS_SUCCESS;
}
```

## [0x02] Notes

| Member | Meaning |
|--------|---------|
| StartVa | Page-aligned start of the virtual buffer |
| ByteOffset | Offset of the real VA inside that page |
